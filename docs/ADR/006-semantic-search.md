# ADR-006: Semantic Photo Search (Post-MVP)

**Статус:** Accepted
**Дата:** 2026-04-01

## Контекст
Пользователь хочет искать фото по текстовому описанию ("девушка в красном платье", "выступление на сцене"). Бюджет: ~$5 на 900 фото.

## Решение
**OpenRouter** (google/gemini-flash-2.0) для генерации описаний → **pgvector** для векторного поиска.

## Обоснование
- gemini-flash-2.0: ~$0.005/фото = $4.50 на 900 фото (в бюджете)
- Описания хранятся как text embeddings в pgvector
- Единая БД для face embeddings и text embeddings
- Post-MVP фича — не блокирует основной функционал

## Альтернативы
| Подход | Плюсы | Минусы | Статус |
|---|---|---|---|
| Локальный CLIP | Бесплатно | Медленно на CPU, менее точные описания | Отклонено |
| Без семантики | Проще | Меньше wow-эффекта | Отклонено |

## Последствия
- API-ключ OpenRouter в конфигурации
- Описания генерируются как отдельный этап pipeline (после кластеризации)
