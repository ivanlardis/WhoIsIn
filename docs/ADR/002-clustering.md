# ADR-002: Face Clustering Algorithm

**Статус:** Accepted
**Дата:** 2026-04-01

## Контекст
Необходимо группировать face embeddings (512-dim ArcFace) в кластеры-персоны. Количество персон заранее неизвестно.

## Решение
**HDBSCAN** (Hierarchical DBSCAN).

## Обоснование
- Не требует ручного подбора eps (главная проблема DBSCAN)
- Автоматически определяет количество кластеров
- Обрабатывает шум: фото без чёткого лица попадают в noise-кластер, а не в случайную группу
- Работает с cosine distance на высокоразмерных данных

## Альтернативы
| Алгоритм | Плюсы | Минусы | Статус |
|---|---|---|---|
| DBSCAN | Проверенный, быстрый | Требует ручной подбор eps — основная боль | Отклонено |
| K-means | Простой | Требует указать K (кол-во персон) заранее | Отклонено |
| Agglomerative | Гибкий | Медленный O(n³) на больших данных | Отклонено |

## Последствия
- Зависимость от hdbscan PyPI пакета
- Порог min_cluster_size нужно подобрать (предварительно: 2-3)
