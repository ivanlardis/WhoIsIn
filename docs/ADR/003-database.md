# ADR-003: Database for Embeddings and Metadata

**Статус:** Accepted
**Дата:** 2026-04-01

## Контекст
Необходимо хранить реляционные данные (мероприятия, фото, персоны) и face embeddings (512-dim векторы) с поддержкой similarity search.

## Решение
**PostgreSQL 16 + pgvector** extension.

## Обоснование
- Единая БД для реляционных данных И векторного поиска
- Cosine similarity на 512-dim векторах нативно через pgvector
- IVFFlat/HNSW индексы для быстрого поиска
- Production-ready, хорошо документирован
- Легко развернуть в Docker

## Альтернативы
| Вариант | Плюсы | Минусы | Статус |
|---|---|---|---|
| SQLite + ручной cosine | Нуль зависимостей | Нет векторного индекса, медленный поиск | Отклонено |
| LanceDB | Специализированное векторное хранилище | Отдельный сервис, лишняя зависимость | Отклонено |
| ChromaDB | Удобный API | Ещё одна зависимость, нет реляционных данных | Отклонено |

## Последствия
- PostgreSQL и pgvector в Docker Compose
- Alembic для миграций
- SQLAlchemy как ORM
